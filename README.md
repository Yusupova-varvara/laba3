Задание №2
Доказуемо простые числа, как правило, используются в качестве модулей криптосистем, основанных на проблеме дискретного логарифмирования, таких как шифр Шамира, криптосистема Эль-Гамаля и связанные с ней стандарты цифровой подписи ГОСТ Р 34.11-94, ГОСТ Р 34.11- 2001, DSA и EСDSA. Рассмотрим тесты на простоту Миллера, Поклингтона, ГОСТ.
Тест Поклингтона
Данный тест основан на теореме Поклингтона.
Теорема Поклингтона. Пусть 𝑛 = 𝑅𝐹 + 1 > 1, где 𝐹 = 𝑞1 𝛼1𝑞2 𝛼2 … 𝑞𝑘 𝛼𝑘, 0 < 𝑅 < 𝐹, qi – различные простые числа, αi – их степени. Тогда n – простое число, если найдется такое целое число a, что выполнены следующие условия: 
1) 𝑎 𝑛−1 ≡ 1 (𝑚𝑜𝑑 𝑛); 
2) НОД(𝑎 𝑛−1 𝑞𝑖 − 1, 𝑛) = 1 для всех 𝑖 = 1, … , 𝑘.

Процедура генерации числа для теста Поклингтона: 
1. Строится таблица простых чисел qi, основанное на решете Эратосфена; 
2. Строится число 𝐹 = 𝑞1 𝛼1𝑞2 𝛼2 … 𝑞𝑘 𝛼𝑘, где qi – различные простые числа из таблицы на шаге 1, αi – их степени (множители и степени выбираются случайным образом). Разрядность числа F должна быть на 1 бит больше половины требуемой разрядности bits; 
3. Вычисляется значение 𝑛 = 𝑅𝐹 + 1, где R – случайное четное число, разрядность которого на 1 бит меньше F (F > R); 
4. Построенное число n отправляется на проверку тестом Поклингтона с заданным параметром надежности t (t > 0). Если после проверки число не является простым, то вернуться на шаг 2 и построить новое число n.
Процедура тестирования: 
Вход: n – число для проверки, 𝐹 = 𝑞1 𝛼1𝑞2 𝛼2 … 𝑞𝑘 𝛼𝑘 – каноническое разложение числа F на множители, t – параметр надежности (t > 0). 
1. Выбрать случайным образом t различных целых чисел aj таких, что 1 < 𝑎𝑗 < 𝑛, 𝑗 = 1, … ,𝑡; 
2. Для каждого aj вычислить 𝑎𝑗 𝑛−1 𝑚𝑜𝑑 𝑛 (малая теорема Ферма). Если какой-либо из результатов не равен «1», то идти на Выход с сообщением «n – составное число».
3. Для каждого 𝑞𝑖, 𝑖 = 1, … , 𝑘, выполнить: 
   3.1. Для каждого 𝑎𝑗, 𝑗 = 1, … ,𝑡 вычислить НОД(𝑎𝑗 𝑛−1 𝑞𝑖 − 1, 𝑛); 
   3.2. Если какой-то из результатов шага 3.1 равен «1» (т.е. числа взаимно просты), то вернуться на шаг 3 и взять следующее 𝑞𝑖. 
   3.3. Если все результаты шага 3.1 не равны «1», то идти на Выход с сообщением «n – вероятно, составное число». 
4. Идти на Выход с сообщением «n – простое число».

Замечания по тесту Поклингтона: 
1. Разрядность чисел, сгенерированных данным тестом, должна совпадать с требуемой разрядностью bits. 
2. Для чисел, отмеченных данным тестом как простые, нужно выполнить дополнительную проверку этим же тестом при параметре надежности 𝑡 = 2. Если число прошло эту проверку, то выдать результат true, иначе false. 
3. Если тест выдал сообщение «n – составное число» или «n – вероятно, составное число», то нужно сгенерировать новое число и увеличить параметр rejected на 1. 
4. Для вычисления НОД нужно воспользоваться алгоритмом Евклида. 
5. Для проверки работы теста можно воспользоваться Приложением А и В. 
6. Для повышения вероятности получения простого числа на этапе генерации числа можно добавить следующие дополнительные проверки: a. НОД(𝑅, 𝐹) = 1; b. 𝐹 > √𝑛 − 1.
Задание №3
Математическая сумма 
      На вход подается два целых числа a и b, каждое из которых находится в диапазоне от 1 до 10 включительно. На основе этих чисел необходимо вычислить сумму заданного бесконечного ряда.
      Если ряд расходится (то есть его сумма стремится к бесконечности), выведите слово infinity. Если ряд сходится, и его сумма является рациональным числом, выведите её в виде несократимой дроби (например, 3/2).
Задание №4
Увлекательная игра
     Павел и Вика придумали увлекательную игру. Сначала они выписывают в тетрадь последовательность из n целых чисел (5 ≤ n ≤ 50 000). Игроки ходят по очереди: на каждом ходу игрок может выбрать от одного до m чисел (2 ≤ m ≤ 100) из начала последовательности, стереть их, и прибавить сумму стёртых чисел к своему счёту. Игрок не может стирать столько же чисел, сколько стёр его противник в предыдущем ходе. 
     Павел ходит первым. Оба игрока действуют по следующей стратегии: на каждом ходу игрок вычисляет префиксные суммы оставшихся чисел и выбирает такое количество чисел (от 1 до m, с учётом ограничения на повторение количества), которое даст ему максимальную сумму. Если несколько вариантов дают одинаковую максимальную сумму, игрок выбирает минимальное возможное количество чисел. 
      Игра заканчивается, когда все числа удалены из тетради, а победителем становится тот, у кого итоговый счёт больше. Гарантируется, что ничья невозможна. 
      Необходимо вывести 1 в случае победы Павла и 0 — в случае победы Вики.
Задание №5
     Задача об остывании чашки кофе в кофейне НГТУ Написать программу, моделирующую процесс остывания кофе. Программа должна позволять задавать все необходимые параметры. Построить графики или таблицы зависимостей величин. Считаем, что температуру кофе измеряли через некоторые, необязательно равномерные промежутки времени. Написать программу, которая бы по данным измерений строила линейную модель, вычисляя статистические критерии их адекватности. 
     Природа переноса тепла от кофе к окружающему пространству сложна и в общем случае включает в себя механизмы конвекции, излучения, испарения и теплопроводности. В том случае, когда разность температур между объектом и окружающей средой не очень велика, скорость изменения Т объекта можно считать пропорциональной этой разности температур. Это утверждение формулируется на языке дифференциальных уравнений так
𝑑𝑇/𝑑𝑡 = −𝑟(𝑇𝑘 − 𝑇𝑠𝑟),
где Тk – температура кофе, Tsr – температура окружающей среды, r – коэффициент остывания. Коэффициент остывания зависит от механизма передачи тепла, площади тела, находящегося в контакте со средой и тепловых свойств самого тела. Знак (–) появляется во избежание нефизического эффекта увеличения температуры тела, когда Tk > Tsr. Данное соотношение называется законом теплопроводности Ньютона. 
В варианте первого курса, необходимо реализовать консоль и таблицу значений (не график). Расчет статистических параметров не является обязательным.
